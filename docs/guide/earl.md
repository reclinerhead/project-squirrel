### The listener (`listener/` — Earl, the ears of the house)

Epic #133 Phase 1 (issue #172): a pearl-resident daemon (`listener/earl.py`) that pulls raw audio from the yard's microphones, runs Cornell's **BirdNET** (the `birdnet` 0.2.x package, `"tf"` backend — 0.2.x dropped tflite) on every 3-second window at 48 kHz, and publishes accepted detections to `audio/events`. Earl is deliberately **domain-agnostic**: he reports what he heard (species, confidence, source, clip path); consumers decide what it means. The first consumer is `listener/sightings.py` — the frame_archiver mold — which turns `audio/events` into the bird record: an append-only `sightings` table plus the **life list** (one row per species ever heard, with its first-heard moment). Phase 0's proof and measurements live in [#133's results comment](https://github.com/reclinerhead/project-squirrel/issues/133); the numbers that shaped this design: inference ~1.2 s per 3 s window per stream via a warm `predict_session()` (real time on one core, so the Coral is a scale play, not a prerequisite), and false positives without region filtering clustered at 0.25–0.61 while true locals scored 0.89–0.92.

- **The YAMNet classifier** (issue #174, Phase 2a): every window meets **YAMNet** (521 AudioSet classes, TF Hub SavedModel, CPU — ~56–80 ms per 3 s window on pearl) before BirdNET. `gate.route()` returns one of three verdicts from a curated, name-verified class map (`earl.py` warns at startup about entries the loaded model doesn't know — the typo'd-constant lesson): **speech** kills the window outright, checked first and at a deliberately *lower* floor (`SPEECH_FLOOR` 0.1, **not configurable**), so the epic's invariant is enforced by a model with real speech classes, with BirdNET's human check remaining as the second layer; **notable** (dog, glass, siren, thunder, engine…) emits a `kind:"sound"` event — coarse class, no species fields, its own `VisitTracker` (class names and species names are different vocabularies), clip and rms and all — *and the window still goes to BirdNET*, because a barking dog doesn't mean no bird is singing; **listen** is everything else. **Only speech withholds a window from BirdNET** — this is the module's central contract, and `MERLE_EARL_GATE_FLOOR` (default 0.3) governs *sound events only*.
  - **Why it is not a gate, in the architectural sense.** It shipped as one on 2026-07-18 — only bird-routed windows reached BirdNET — and ate **17 of 25 confirmed yard detections** in an afternoon (a 0.97 pheasant, a 0.88 waxwing, a 0.86 nighthawk), because (a) it was desk-validated on a close-mic'd clip scoring `Bird=0.52` while real distant yard birds score a *median of 0.189*, several with no bird class in the top-7 at all, and (b) AudioSet is a **hierarchy**: on faint audio the parent classes fire hardest (`Animal=0.505` where `Bird=0.463`), and the map had been built from plausible-sounding names rather than measured behaviour. `Animal`/`Wild animals` accordingly moved into `BIRD_CLASSES` (reporting only now) and out of `NOTABLE_CLASSES`. The deeper error was accepting a veto at all: gating BirdNET was justified solely by CPU savings, and pearl idles at load 0.00 running two ears — the premise was dead before the code shipped. `test_listener_gate.py` now carries the real YAMNet scores of those confirmed birds as regression fixtures.
  - Per-worker counters log every 100 windows (`gate stats: … listen, speech-killed, notable, ungated; gate N ms avg`). A classifier that can't load runs Earl **ungated** (Phase 1 posture) loudly. The YAMNet call sits behind one function (`load_yamnet()`'s `classify`) — **the 2b seam**: the Coral phase swaps its implementation and nothing upstream moves. Audio reaches it at 16 kHz via mean-of-3 decimation; BirdNET always sees full-rate. `sightings.py` needed zero changes — its ignore-unknown-kinds guard (#172) is the two-tier schema's compatibility mechanism, working as designed.
- **The gate rules** (`listener/gate.py`, pure — every rule under `test_listener_gate.py`): (1) **speech is never an event** — a window whose top label is Human-anything produces no event and no clip, by construction, not by setting (epic principle 1: Earl classifies, he never records conversations); lower-ranked human labels are dropped individually. (2) **The region mask (D4, mandatory)**: BirdNET's geo model, fed `MERLE_LATLON` + the BirdNET week (48-week year, 4/month — `week_of()`), yields the locally-plausible species set; everything else is discarded. Recomputed when the week ticks; a geo-model failure runs **unmasked and loud**, never silently eventless. (3) **The threshold** (`MERLE_EARL_THRESHOLD`, default 0.65 — between Phase 0's false-positive ceiling and true-local floor), which wind **raises but never lowers**: above 15 mph the bar becomes 0.75 and survivors carry `wind_suspect: true`. Wind arrives from the retained `weather/current` on Earl's own MQTT connection (the narrator's staleness rule: a report older than 30 min degrades to calm-day rules — the Ecowitt earns its keep again).
- **Process model — shaped by the fork trap.** `birdnet`'s predict calls fork worker children, and forking while any thread holds a lock (paho's network thread qualifies) deadlocks the child **silently** — Phase 0 lost a debugging cycle to this. So the parent (`Earl`) owns all MQTT and never loads birdnet; each source runs in its own **spawned** worker process (fresh interpreter, no inherited locks, `daemon=False` — daemonic processes can't have children, and birdnet's session needs children) that stays single-threaded around its predict session and reads its capture pipe with bare `os.read` (no buffered-reader lock). Workers report over a spawn-context queue; wind rides a shared `Value` the other way. A dead capture restarts with backoff (3/10/30/60 s) forever; a dead worker is respawned by the parent; neither kills Earl.
  - **Stalls, not just exits** (issue #201): that restart loop only ever fired on a capture that *ended*, and the field's real failure is a capture that **hangs while still running** — the Amcrest unplugged and moved mid-session sends no TCP reset, so the socket stays `ESTAB` and ffmpeg blocks on it forever; merle powered down to recharge half-opens the ssh, whose kernel keepalive is ~2 hours away. Either way the source dies **silently** — no log line, no state change, the service healthy-looking — which is precisely how one went unnoticed for an hour on 2026-07-19. Three layers, and the first two exist to turn a stall into an exit so there is exactly **one** path back online rather than a second parallel recovery: (1) ffmpeg's `-timeout` (the *rtsp demuxer's* socket-I/O bar, in **microseconds** — **not** `-rw_timeout`, which that demuxer does not carry; measured on pearl's ffmpeg 8.0.1 against a socket that accepts and never speaks, it exits in ~9 s where the unflagged run was still waiting at 20 s); (2) `ServerAliveInterval=5`/`ServerAliveCountMax=3` on the rover's ssh; (3) `read_exact`'s **`select` timeout**, the source-agnostic backstop, which gives the read a third answer — `STALLED`, a sentinel object distinct from EOF's `None` because the caller's whole branch is an `is` check. It is `select` on the fd and **not a watchdog thread** for the same reason the worker is single-threaded at all: a thread here is the fork trap. The bar is **bytes, not windows** — the clock restarts on every chunk, so only a source that has gone completely quiet for `STALL_TIMEOUT_S` (15 s, five missed windows) is called stalled, and one dribbling slower than the timeout still keeps its window. A stall kills the capture and falls into the same backoff, logging `capture stalled` rather than `capture ended` so the journal distinguishes the two. Net field behavior, which is the actual acceptance bar: **a device can be gone for hours and its source resumes within ~60 s of coming back, unattended** — the backoff cap, since a window that arrives resets `backoff` to 0.
- **Sources** produce s16le/48 kHz/mono on stdout. `amcrest`: ffmpeg audio-only RTSP (`-allowed_media_types audio` — the 48 kHz AAC track alone, none of the 4K video's bandwidth; camera-side audio must be enabled in the cam UI, done 2026-07-18). `rover`: `arecord` on merle piped over ssh (the USB camera's own mic, `plughw:0,0` — Phase 0 D5: the JMTek dongle is silent), the whole command overridable via `MERLE_EARL_ROVER_CMD`; requires pearl→merle ssh keys (a deploy step). Per-source health is **state** on the retained `audio/sources` JSON (`starting`/`online`/`silent`/`offline` + last-window ts, republished on change — `silent` means ~1 min of rms-nothing, a mic that died upstream of the codec); `audio/status` itself stays a **raw** `online`/`offline` string with Last Will, the `weather/status` contract verbatim. (The epic sketched per-source detail *in* the status payload; the house convention — status topics are raw strings consumers render verbatim — won, and the detail moved to its own topic.)
- **Visits — the species-level debounce** (issue #175, `gate.VisitTracker`, pure): day one measured **3.2× event redundancy** (one singing cardinal = 25 events/rows/clips) and a disk burn 10× the deploy estimate — the `SpeciesPresence` failure mode, one sense over. A *visit* opens on a detection of species S with no S-detection in the preceding `VISIT_GAP_S` (60 s) and stays open while they keep arriving; only the **opening** detection publishes (liveness: consumers hear the arrival in seconds, nothing waits for the bird to stop). Later windows are suppressed, except that a window beating the visit's best confidence **rewrites the visit's clip in place** — the published clip path never dangles, and a lifer's `first_clip` ends up holding the bird's best moment. Visit close is a lazy sweep on the worker's own thread (no timers — the fork-trap rule) that logs the stats (`visit closed: Northern Cardinal, 25 windows, 96s, best 0.90`); the journal is where a visit's true best lives, since the published event carried the opening window's confidence (append-only store, no UPDATE path — accepted trade-off). Visits deliberately survive capture restarts (a reconnect inside the gap continues the visit); species interleave independently; sources stay share-nothing.
- **Clips (D1: files + paths)**: a per-source ring of raw windows; a visit-opening (or best-upgrading) window is written as prev+event+next (~9 s of context) under `MERLE_EARL_CLIPS/<source>/<epoch>-<species>.wav` (allowlist-scrubbed names — the frame_archiver rule), and the event carries the relative path. **One clip per visit** (#175), not per window. A failed write is a clip-less event, never a dead worker. `publish_bytes()` stays the frames' idiom — clips are 10–100× frame size and both ends live on pearl.
- **The bird record** (`listener/sightings.py`, own unit): `sightings` is append-only with an id key (two species can share a window's ts) and, since #175, an `rms REAL` column — the window's raw signal level, persisted because **confidence is model certainty, not loudness**, and Phase 5's "heard strongest at the east cam" ranking needs the real number (day one: rover louder in 7 of 9 same-window pairs). `connect()` adds the column to a pre-#175 file idempotently (the repeatable-pass rule — an upgrade is a restart, not a migration script; old rows honestly NULL). `life_list` uses `INSERT OR IGNORE` so **first-heard is enforced by the store** — replays and restarts can never move a `first_ts`. Epoch-seconds timestamps (the audio namespace is epoch end to end, the weather reasoning). `MERLE_EARL_DB` (default `earl.db` under `WorkingDirectory`) is the **third irreplaceable file** the stack owns, after `weather.db` and `music.db`'s ratings: a first-heard date is a moment no API sells back. A sighting missed while the consumer restarts is a bus moment nobody archived — but the clip Earl wrote is still on disk, so a future backfill pass over `clips/` can heal the record (the enrichment-pass ethos).
- **Clip retention** (#175) lives in `sightings.py`, not `earl.py` — deliberate: the store knows which clips are irreplaceable, so the store prunes. Hourly (first pass at startup, so a long-down consumer heals immediately), on the main thread beside paho's loop, with its own short-lived read connection: files older than `MERLE_EARL_CLIPS_KEEP_DAYS` (default 90 — ~20 GB steady-state post-debounce) are removed, **except every path in `life_list.first_clip`, exempt forever** — a lifer's first recording is part of the permanent record. Selection is pure (`prune_selection()`, injected clock + listing + exempt set — the `frame_archiver` precedent). Pruned rows keep their `clip` path: a missing file is an honest gap, the Field Journal's pruned-thumbnail convention. **Enhanced siblings (#190) share their original's fate exactly** — a doomed clip takes its `-enh.wav` with it, and a lifer's exempt clip keeps its sibling. Judging a sibling on its own age would be wrong in both directions: it is always the newer file (a later pass wrote it), so it would outlive the evidence it enhances, and an exempt lifer's sibling would age out from under a permanent clip. An orphaned sibling — original already gone — has nothing to inherit from and ages out on its own. The suffix is duplicated as a two-line constant here rather than imported: `clip_enhance` needs numpy and this unit runs from the lean repo venv, which `test_import_boundary.py` enforces.
- **Clip enhancement** (`listener/clip_enhance.py`, issue #190): the pass that makes a faint bird *audible*. A large share of real clips are birds buried under a plane, road noise, wind, or HVAC — BirdNET finds them anyway, but the clip is the only evidence a human can independently evaluate, and a high confidence on a barely-audible signal is exactly the profile of a plausible false positive. Being unable to hear the clip means being unable to check the model. Three steps, and **the order is the design**: (1) **band-limit** — songbird song lives ~2–8 kHz while the interferers are overwhelmingly sub-1 kHz, so a rolloff below 1.5 kHz (raised-cosine, not a brick wall, which rings) removes most of the noise and almost none of the bird; (2) **spectral subtraction using the clip's own pre-roll as the noise profile** — the key move, and nearly free, because `write_clip()` already stores prev+event+next, making the leading 3 s *this exact acoustic scene without the bird in it*, a perfectly matched per-clip estimate; subtraction uses a proportional spectral floor, since subtracting to zero is what produces musical noise; (3) **normalize, last** — normalizing first would normalize *the plane* and leave the bird just as faint in relative terms. Measured on a synthetic 9 s clip (0.9-amplitude rumble, 0.012 warble): event-window level vs pre-roll goes from 0.97 (indistinguishable) to the bird alone at full scale. **numpy only** — `numpy.fft` covers both the FFT-domain band and the STFT, and scipy would fight the lean-venv posture for no gain. The STFT pads both edges by a full window overlap: without it the first samples sit under a single Hann taper whose overlap-add divisor approaches zero, which turns the clip's opening into a 300× spike *the moment the spectrum is modified* — it round-trips fine unmodified, so the trap only springs once the pass does its job. **Never in place**: a later `best` verdict rewrites clips, so the pass writes a sibling `<stem>-enh.wav` and the original is never modified or deleted. Idempotence falls out of mtime — a sibling newer than its original is skipped, and a rewritten clip is newer than its sibling and gets re-enhanced for free, so file existence is the source of truth and no schema changed. Worklist-driven with `--refresh`/`--limit`/`--dry-run` (the `species_profile` shape; a future per-clip re-enhance button calls what `--refresh` calls). **The pass is not automatic** — pearl auto-deploys merged code but never runs passes.
- **Deployment**: Earl runs from his **own Python 3.11 venv** on pearl (`~/earl-venv` — BirdNET's TensorFlow has no wheels for the repo venv's Python; built with `uv`), as the `earl-listener` unit; `sightings` needs only paho and runs from the **repo venv** as `earl-sightings`. Clips land on the media-cache LV (`/srv/media-cache/earl`). Setup and unit files: `Servers/Pearl.md`. `requirements.txt` is untouched — birdnet lives only in Earl's venv, and `test_import_boundary.py` now asserts the whole `listener` package imports clean under poison (earl.py's heavy imports are lazy, inside the worker functions), so the lean sightings unit can't die reaching for TensorFlow.
- Pure logic (the gate's three rules, the visit lifecycle and best-clip upgrades, event shaping incl. rms, clip-path scrubbing, `MERLE_LATLON` parsing, week math, source-command parsing with redacted RTSP argv, sightings row parsing/recording/first-heard, the rms column upgrade, prune selection with the lifer exemption and enhanced siblings) is covered by `test_listener_gate.py` + `test_listener_sightings.py`; the enhancement DSP (band response at 200 Hz vs 4 kHz, subtraction against known noise, STFT round-trip, **the normalize-ordering assertion**, mtime idempotence, worklist arms) by `test_listener_clip_enhance.py` against synthesized signals rather than real clips; and **the capture read's three answers** — window, EOF, `STALLED`, plus the false-trip guard that a slow-but-alive source keeps its window and the assertion that both capture commands carry their own timeouts — by `test_listener_earl.py` against real `os.pipe()` fds. That last file **skips on Windows, in the fixture** so future pipe tests inherit it: `select` accepts only sockets there, and reshaping a pearl daemon's read around a platform it never runs on would be the tail wagging the dog. The capture loops, BirdNET sessions, WAV I/O, and MQTT plumbing are desk-tested against the real camera, broker, and yard.
