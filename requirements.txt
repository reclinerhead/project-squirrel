# Merle Python dependencies.
#
# The vision stack (live.py, prelabel.py, dedup.py) also needs ultralytics +
# torch, which are large, GPU-specific, and installed out-of-band per machine --
# so they're intentionally NOT pinned here. This file covers the daemon runtime
# and the shared vision libs.
#
# Note on OpenCV: the daemon and live.py use full opencv-python (live.py's
# imshow needs the GUI build). CI installs opencv-python-headless instead, since
# it only runs the headless daemon tests -- see .github/workflows/tests.yml.

fastapi
uvicorn[standard]
opencv-python
numpy
paho-mqtt      # the MQTT bus (daemon publisher, narrator, replay script)
pyyaml         # narrator personas + character bible

# Tag reading for the music indexer (music_index.py). Optional by design: the
# audio-span parsing and hashing are pure stdlib, so a box without mutagen
# still produces a complete, correctly-identified catalog -- just with null
# tags. Not needed in CI, which only exercises the pure half.
mutagen

# Audio analysis for the Phase 1a backfill (jukebox/music_analyze.py). Like
# ultralytics/torch above, this is machine-specific and installed out-of-band:
# it's needed ONLY on bluejay, only for a one-time pass, and it drags in numba +
# llvmlite + scikit-learn (~100 MB) that pearl and CI have no use for. The
# analyzer imports it lazily inside estimate_bpm() so its pure half stays
# importable -- and testable -- without it.
#
#   pip install librosa      # bluejay only, and only for the backfill
#
# It also needs ffmpeg on PATH, which is not a pip package: the library is 62%
# ALAC (libsndfile can't decode it) and ebur128 is ffmpeg's loudness filter.
#   winget install Gyan.FFmpeg

# Test-only:
pytest
httpx
